# MirrorDefender · M6 操作与 UI 大版本需求与开发计划

> 状态：**需求已确认，进入实现**。日期：2026-07-22。  
> 文档职责：本文件是 M6 的长期事实源，固定交互语义、整体 UI 布局、数据边界、批次顺序和验收标准。实现细节同时回写对应 `Docs/systems/` 文档。  
> 里程碑调整：旧版路线中的“反射镜 M6 / UI M7”已被本次决策替代；当前 **M6 = 操作与 UI 大版本**，反射镜顺延到后续待排期里程碑，不视为取消。

---

## 一、版本目标与边界

### 1.1 目标

1. 用正式卡槽替换开发期建造面板，建立清晰、可扩展的左键放置流程。
2. 建立正式运行时 HUD：建筑卡槽、地块详情、全局信息、经济信息、暂停菜单、波次时间轴、倍速与战术慢放。
3. 为每关提供 6 个可编辑镜头预设，并支持数字键平滑切换。
4. 将散落的运行时调试文字和控制入口统一迁移到游戏内控制台。
5. 所有 UI 美术、图标和卡面预留资源接口；无正式资产时使用统一的镜框/镜面/玻璃灰盒。
6. 保持运行时 HUD、交互控制、时间控制、数据模型相互解耦，避免继续扩张 `Main.gd` 和开发期 `M3DebugPanel`。

### 1.2 不改变的内容

- 不改变现有塔、屏障、边障、镜子、虚像、地块元素、敌人、路径和波次玩法。
- 不调整建筑/敌人/资源等既有数值；唯一新增的时间行为是用户明确要求的战术慢放、暂停和 2 倍速。
- 不替换现有 3D 关卡画面和世界内悬浮操作，只为其接入正式交互状态并统一视觉风格。
- 边障不进入默认携带卡组，但其既有玩法保留，并可通过调试控制台继续测试。
- 本阶段不实现正式美术资产；接口、占位视觉和替换路径必须完整。

## 二、整体 UI 布局（长期结构约束）

### 2.1 运行时画面锚点

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ ┌──────────────┐                                      ┌───────────────────┐ │
│ │ 未来波次 ↑   │                                      │ 关卡名 / 据点 HP │ │
│ │              │              3D 关卡                 │ 敌人 / 建筑 / 镜子│ │
│ │ 当前波次     │                                      └───────────────────┘ │
│ │              │                                      ┌───────────────────┐ │
│ │              │                                      │ 地块详情（按需）  │ │
│ │              │                                      │ 多条目滚动展示    │ │
│ ├─当前时间线──┤                                      └───────────────────┘ │
│ │ 首波启动按钮 │                                                               │
│ └──────────────┘                                                               │
│                    ┌镜子┐┌卡1┐┌卡2┐┌卡3┐┌空┐┌空┐┌空┐  金币 慢放 2x 暂停 │
└──────────────────────────────────────────────────────────────────────────────┘
                              ┌──────────────────────────┐
                              │ 暂停菜单 / 调试控制台层  │
                              └──────────────────────────┘
```

### 2.2 固定区域

| 区域 | 锚点 | 内容 | 展开规则 |
|---|---|---|---|
| 波次时间轴 | 左侧，纵向 | 未来/当前波次、当前时间线、首波启动按钮 | 常驻；无波次时显示空态 |
| 卡槽 | 正下方居中，单行 | 独立镜子槽 + 可调数量的建筑槽 | 常驻；空位显示空镜面 |
| 全局信息 | 右上角 | 关卡名、据点生命、场上敌人、建筑/镜子上限 | 常驻 |
| 地块详情 | 右侧中部 | 地块示意与全部内容条目 | 未选中或空地块时收起 |
| 经济与时间 | 右下角 | 资源、自动慢放、1x/2x、暂停 | 常驻 |
| 暂停菜单 | 屏幕中央模态层 | 设置、重启、退出 | 暂停后出现并拦截世界输入 |
| 调试控制台 | 覆盖层 | 分类开关、日志、命令输入 | F1 开关并拦截世界输入 |
| 世界悬浮操作 | 选中实体附近 | 删除、升级、旋转；镜子删除、翻面 | 仅选中对应实体时出现 |

### 2.3 响应式与层级

- 基准布局为 1600×900，同时验收 1280×720、1920×1080。
- 卡槽必须始终保持单行；空间不足时等比缩小至最小卡宽，不换行。
- 地块详情使用滚动容器管理任意数量条目，不能撑出视口。
- UI 点击必须被 UI 消费，不得穿透到世界地块/边选择。
- 推荐 CanvasLayer 层级：世界 HUD < 浮动提示 < 暂停模态 < 调试控制台。

## 三、输入与交互事实源

### 3.1 鼠标规则

- 左键承担所有肯定操作：选卡、选地块/边、放置、实体操作按钮、HUD 按钮。
- 右键只承担取消：清除卡片选择、放置预览、世界选择和地块详情，回到选择模式。
- UI 控件获取鼠标事件后必须阻止世界选择与放置。
- F1 控制台输入期间，屏蔽世界输入、镜头预设和卡槽快捷键。

### 3.2 交互状态

正式状态由 `RuntimeInteractionController` 管理；`M3DebugPanel` 不再是运行时交互事实源。

| 状态 | 进入方式 | 左键世界行为 | 退出方式 |
|---|---|---|---|
| `SELECT` | 默认/取消/任一放置尝试完成 | 选择非空地块并展开详情；选中实体时显示悬浮操作 | 选择卡片进入对应放置状态 |
| `PLACE_BLOCK` | 选择块建筑卡 | 尝试在格上放置一次 | 无论成功或失败，立即回 `SELECT` |
| `PLACE_EDGE` | 选择边建筑卡 | 尝试在边上放置一次 | 无论成功或失败，立即回 `SELECT` |
| `PLACE_MIRROR` | 选择独立镜子卡 | 预览并尝试在边上放置一次 | 无论成功或失败，立即回 `SELECT` |

### 3.3 单次放置契约

一次卡片选择只允许一次世界放置尝试：

1. 成功：完成扣费和放置，取消卡片选择、预览与建造状态，恢复选择模式和正常时间；不自动选中新建筑。
2. 资源不足：卡片通常已置灰不可选；若选中后资源变化导致不足，点击仍视为一次失败尝试并退出。
3. 达到上限：同上，失败后退出。
4. 地块/边不可放置、被占用、未命中合法格/边：显示失败原因并退出。
5. 右键取消不计为放置尝试，不产生资源变化。

### 3.4 选择模式

- 点击空地块：清除实体选择并收起地块详情。
- 点击非空地块：打开右侧详情；若含可操作实体建筑，同时保留现有世界悬浮操作。
- 选中实体建筑、实体边建筑或实体镜子时进入战术慢放；选中虚像和单纯查看地块不进入慢放。
- 右键取消当前选择并恢复正常时间。

## 四、卡槽与携带配置

### 4.1 默认结构

```text
复制镜（独立，不计建筑卡槽数） | 箭塔 | 激光塔 | 屏障 | 空槽 | 空槽 | 空槽
```

- 建筑卡槽数量是关卡/运行时可调参数，默认 `6`。
- 默认仅携带箭塔、激光塔、屏障三种块建筑。
- 复制镜固定在最左侧独立槽；它计镜子上限，不计建筑携带槽数。
- 空位显示为空镜面，不能选择。
- 数字快捷键保留给镜头预设，不用于卡槽。

### 4.2 卡片表现与可用性

- 纵向长方形；外框模拟镜框，卡面模拟镜面/玻璃。
- 主体显示建筑图像，底部显示当前等级建造费用。
- 资源不足或对应建筑/镜子达到上限时全卡置灰且不可选择。
- 已选卡增加一圈清晰金色外框。
- 资源、数量或上限改变时即时刷新可用性；如果已选卡变得不可用，保留到下一次点击并按失败尝试退出，或由控制器主动取消并明确提示，两种实现只能选一种并写测试。M6 默认采用前者以保持“点击一次即结束”的可预期性。

## 五、地块详细信息

### 5.1 展开条件

“非空地块”包含以下任一内容：

- 实体块建筑；
- 与该地块相邻边上的实体边建筑或镜子；
- 同格重叠的全部虚像；
- 大石头、尖刺、黑洞等关卡元素；
- 关卡元素的运行时状态，例如石头耐久、黑洞装填量。

### 5.2 信息模型

面板按条目动态生成并滚动。每条至少预留：

- 名称、类型、图标/预览资产；
- 实体/虚像；
- 虚像根源地块与产生镜子；
- 建筑等级、当前/最大耐久；
- 建造朝向或边方向；
- 关卡元素的核心运行时数据；
- 后续扩展字段列表。

面板只读；删除、升级、旋转、镜子翻面继续使用世界悬浮按钮。

## 六、时间、经济与暂停

### 6.1 时间倍率优先级

```text
暂停 0x  >  自动战术慢放 0.1x  >  用户快速 2x  >  正常 1x
```

- 自动战术慢放倍率可调，默认 `0.1`；右下按钮仅控制“自动慢放是否启用”。
- 选择建筑/镜子卡或实体建筑/实体边建筑/实体镜子时触发；取消、完成放置或清除实体选择后恢复下一级倍率。
- 2x 按钮在 1x/2x 间切换；慢放期间仍记住用户选择的 2x，退出慢放后回到 2x。
- 暂停期间记住暂停前状态，关闭暂停后按当前选择和快速状态重新求值。
- 战斗、波次、经济和敌人移动使用统一缩放后的模拟时间；HUD 数字动画、弹字、控制台、暂停菜单使用不受缩放的 UI 时间。
- 战术慢放期间镜头操控保持正常手感，不随 0.1x 变慢。

### 6.2 经济显示

- 右下显示当前资源。
- 资源变化时，显示金黄色 `+x` / `-x`，从资源栏向上移动并淡出。
- 主数字在短时间内滚动/插值到新值，不能瞬间跳变。
- 连续变化事件分别产生弹字；主数字动画从当前显示值平滑追赶最新真实值。

### 6.3 暂停菜单

- 暂停按钮打开居中模态菜单，游戏模拟为 0x。
- 从左到右：设置、重启关卡、退出游戏。
- 首批设置项：主音量、全屏/窗口、UI 缩放；保存到 `user://settings.cfg`。
- 重启关卡必须深度重载当前 LevelResource 的全套运行时状态。
- 退出游戏调用安全退出；暂停菜单和确认交互不向世界穿透。

## 七、全局信息

右上常驻展示：

1. 关卡显示名；
2. 据点当前/最大生命；
3. 当前场上存活敌人数；
4. 实体建筑数量/建筑上限；
5. 实体镜子数量/镜子上限。

虚像不计实体建筑和镜子上限。数据必须来自对应 Manager 信号或查询接口，HUD 不自行维护第二份玩法状态。

## 八、敌人波次纵向时间轴

### 8.1 时间语义

- 最下方横线表示当前战斗时间；未来波次在上方并随时间向下移动。
- 一个波次块的计划时间取该波全部 `SpawnGroup.start_delay` 的最小值。
- 现有 `start_delay` 语义不变：全部相对首次点击“开始第一波”的全局时间原点。
- 波次块下边触及当前时间线时表示该波开始。
- 首波启动按钮位于时间轴最下方，首次点击后建立时间原点；不额外改变后续波次自动开始规则。

### 8.2 悬停与路径预览

- 悬停波次块显示该波全部敌人组成、数量、出怪组和路径。
- 收集该波涉及的全部唯一 Path；每条路径同时从出生点到据点播放快速、循环、发光的方向流线。
- 离开悬停、波次块销毁、暂停菜单或控制台覆盖时清理预览。
- 预览是纯表现，不改变 PathManager、敌人寻路或波次状态。

## 九、六个关卡镜头预设

### 9.1 运行时

- 数字键 `1`～`6` 对应 6 个关卡独立机位。
- 每个机位保存：焦点世界坐标、yaw、pitch、缩放距离。
- 切换使用可调平滑时间，默认 `0.35s`；未配置的槽位无动作。
- 手动 WASD/QE/XC/滚轮输入可在过渡后继续调整；控制台输入期间不响应数字键。

### 9.2 关卡编辑器

- `LevelResource` 持有 6 个可选镜头预设资源。
- 关卡编辑器新增独立“镜头”页/组件，不继续堆积到已有 1300 行级面板脚本。
- 支持从当前编辑视角写入槽位、跳转预览、清空槽位，并显示是否已配置。

## 十、游戏内调试控制台

### 10.1 打开与分类

- `F1` 打开/关闭；打开时捕获键盘与鼠标。
- 可运行时选择显示分类：`grid`、`pick`、`path`、`reroute`、`mirror`、`combat`、`fps`、`wave`。
- 现有 `LevelDebugPanel`、`M3DebugPanel` 和 `Main` 上的调试文字逐步迁入；最终正式 HUD 不显示散落调试信息。

### 10.2 命令注册表

首批命令：

- `help`、`clear`；
- `reload`、`load <res://...>`；
- `resource add <n>`、`resource set <n>`；
- `wave start`；
- `spawn <enemy_id> [path_id]`；
- `debug list`、`debug set <category> <on|off>`。

命令通过注册表扩展，不在控制台 UI 内写大型 `match`。命令返回结构化成功/失败消息，参数错误不得导致运行时报错。

## 十一、模块化架构

### 11.1 推荐模块

| 模块 | 职责 | 禁止承担 |
|---|---|---|
| `RuntimeInteractionController` | 模式、卡片选择、单次放置、取消、世界选择协调 | 绘制卡片、保存资源数值 |
| `GameTimeController` | 正常/快速/慢放/暂停的优先级与 `Engine.time_scale` | UI 具体按钮样式 |
| `RuntimeHud` | 组合各 HUD 子场景并连接公共信号 | 复制玩法 Manager 状态 |
| `BuildCardBar` | 卡槽视图、可用性表现、选卡信号 | 直接放置建筑 |
| `TileInspectionService` | 将格上实体、边实体、虚像、元素转换为只读模型 | 修改建筑/地块 |
| `TileInspectorPanel` | 展示检视模型 | 查询世界树实现玩法判断 |
| `GlobalInfoPanel` | 展示全局只读数据 | 自维护敌人/建筑计数 |
| `EconomyPanel` | 资源滚动数字和增减弹字 | 改资源 |
| `PauseMenu` | 模态 UI 和设置界面 | 实现关卡装配 |
| `WaveTimelineModel/Panel` | 计划时间模型和纵向表现 | 改写波次调度 |
| `PathHoverPreview` | 波次悬停路径流线 | 修改手工路径 |
| `CameraPresetDefinition/Controller` | 关卡数据、平滑切换 | 拦截普通相机输入 |
| `DebugCommandRegistry/Console` | 分类、命令注册与执行 | 把业务命令硬编码在 UI |

### 11.2 数据与依赖边界

- `Main.gd` 只负责装配和高层生命周期，不继续承接具体 HUD 交互。
- UI 通过信号、Callable 和只读 ViewModel 连接 Manager；Manager 不依赖具体 Panel 类。
- 开发期面板可作为兼容适配层，但正式状态和数据只能有一个事实源。
- 每个 UI 子系统使用独立 `.tscn + .gd`，公共镜面风格进入 Theme/StyleBox/Shader 资源。
- 所有动态连接必须在释放/切关时断开，重复装配不得叠加信号。

## 十二、美术与资源接口

至少预留以下可替换资源：

- 全局 UI Theme；卡片镜框、镜面、金色选中框、置灰材质、空镜面；
- 建筑卡图、镜子卡图、地块元素图、敌人图；
- 金币图标、慢放、倍速、暂停、设置、重启、退出图标；
- 波次块底图、当前时间线、首波按钮、路径流线材质；
- 地块详情条目底板与实体/虚像标签；
- 暂停遮罩、面板和按钮；控制台面板。

数据资源应提供 `ui_icon`/`card_icon` 等可选字段：`BuildingDefinition`、`CopyMirrorDefinition`、`TileDefinition`、`EnemyDefinition`。未赋值时使用稳定灰盒，不允许空资源导致脚本错误。

## 十三、批次计划

### 批次 1：卡槽、单次放置与战术慢放基础（已完成，2026-07-22）

**范围**：

- 正式底部卡槽：独立镜子槽、默认 6 建筑槽、三张默认卡、空镜面、费用、置灰、金框和图标接口。
- `RuntimeInteractionController` 接管选择/放置；成功和所有失败均只尝试一次并回 `SELECT`。
- 右键统一取消；UI 输入不穿透。
- `GameTimeController` 实现 0x/0.1x/1x/2x 优先级基础；批次 1 交付自动慢放按钮，暂停菜单和 2x 正式视觉留到批次 3。
- 正式运行时隐藏旧 M3 建造模式控件，保留必要的调试兼容入口。
