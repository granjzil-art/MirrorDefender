# 镜子系统 · Mirror（核心）

> 模块职责：管理复制镜、反射镜两类镜子的放置、朝向、生效逻辑。初版仅这两类。
> 关键架构 / 函数索引 / 参数在文末维护，改代码即改此文档。
> 状态：详细设计 **已确认（2026-07-16 二次拍板）**

---

## 一、通用规则（两类镜子共享）

1. **放置**：镜子挂在**地块的边(Edge)**上，严格贴合。六边形每格 6 条边，正方形每格 4 条边。
2. **不受地块限制**：任意边都可放（含路面、障碍格的边）。
3. **镜长 = 边长**：镜子完整覆盖整条边，可视作"这条边变成了一段镜面"。**不存在半条边的镜子**。
4. **空间占用（已确认）**：镜子对**这条边本身**有占用效果——**一条边至多放一面镜子**。因为一条边被两个相邻格共享，所以两格中任一格从这条边放镜子都会占用它，另一侧不能再放。放置检查以"边"为唯一键（`edge_ref`）。
5. **单面朝向**：镜子有正反两面，只有**生效面**朝向的一侧参与计算。
   - 六边形边镜 → 6 种朝向（每条边一个法线方向）。
   - 正方形边镜 → 4 种朝向。
6. **朝向旋转**：放置后可用 `R` 键在该边的两个法线方向间翻面（边只有正反两侧，翻面=切换生效侧）。
   > 注意：镜子"朝向"本质是**边的哪一侧生效**，只有 2 个状态（正/反），不是 6 个。"6 种朝向"指的是镜子可放在 6 条不同的边上，每条边各有正反。放置时先选边（决定对称轴），再翻面（决定生效侧）。
7. **上限**：镜子数量受资源系统的 `mirror_cap` 限制，与建筑共用主资源。
8. **对称轴 = 镜子所在边所在的直线**（这是所有镜像计算的几何基础）。
9. **鼠标交互（已确认）**：选中/悬停到某个格子或某条边时**高亮显示**（格高亮 + 边高亮两套 gizmo），便于精确贴边放置。

### 边与法线定义（实现约定）
- 每条边用 `(tile, edge_index)` 标识。六边形 edge_index ∈ [0,5]，正方形 ∈ [0,3]。
- 每条边有一条**边中线（对称轴）**和两侧法线 `+n / -n`。
- 生效侧记为 `active_side ∈ {+, -}`，`R` 键翻转它。

## 二、复制镜 · CopyMirror

### 2.1 行为定义
- 当镜子**生效面朝向**某个地块（即该地块在生效侧），把**该地块内的建筑**沿对称轴镜像，生成一个**投影镜像**到对称位置。
- **六边形上边的镜子只有朝下才复制下方地块建筑** —— 即生效侧必须指向"有建筑的那个地块"。

### 2.2 对称轴与镜像位置（关键几何）
- 对称轴 = 镜子所在边所在直线。
- 镜像位置 = 原建筑相对该直线的**镜面对称点**。
- **不同边（不同朝向）→ 不同对称轴 → 镜像落点不同**。这正是玩法深度来源。

```
   [A格: 有塔]          镜子在A/B公共边上, 生效侧朝A
   ────●──── ← 对称轴(边)   原塔在A, 镜像落在B(对称点)
   [B格: 空]           塔的朝向/位置沿轴翻转
```

> 六边形几何提示：六边形相邻两格关于公共边**并非简单上下翻转**，对称轴是斜的。实现时统一用"点关于直线镜像"的通用公式，不要对每条边写特例。

### 2.3 投影镜像的性质（严格遵守用户定义 · 2026-07-16 修正）
- **占格子（修正）**：镜像**占据落点所在的格子**（视觉上是一个投影建筑）。它不可被点选为独立目标、不参与索敌，但**占位**——这带来下面的生成约束。
- **无独立 AI / 无独立索敌**：不自己找目标。
- **无独立动画时钟**：动画、攻击时机、行为**完全复制原件**（经镜像变换后）。
- **联动开火**：**原件索敌并攻击时，镜像才同步攻击**（对镜像空间中的对应目标）。
- 原件消失 → 镜像立即消失。

### 2.4 攻击朝向的镜像（已确认 · 允许打空）
- 镜像**不索敌**。它的**攻击朝向完全由"原实体朝向"经镜像变换得到**：`镜像朝向 = reflect(原件朝向, 对称轴)`。
- 因为镜像朝向是纯几何镜像、不重新索敌，所以**镜像空间里对应位置可能没有敌人 → 会出现"打空"**。这是**符合预期的设计**（镜像是原件的忠实倒影，不是第二个智能单位）。
- 攻击落点 = 原件落点关于对称轴的镜像点。

### 2.4b 生成约束（已确认 · 关键改动）
镜像**占格**，因此生成时严格校验落点格：
- 落点在**地图外** → **不生成**镜像。
- 落点格被 **建筑 / 其它镜像 / 障碍物 / 地形（隆起遮挡等占位）** 占据 → **不生成**镜像。
- 即：**只有落点格为空且在图内时才生成**。原件条件恢复满足（落点变空）时按当前逻辑重新尝试生成。

### 2.5 预留接口（用户要求）
- `ICopyable` 接口：初版只有 `Building` 实现；后续 `Obstacle`(石头) 等可实现，用于"复制障碍物挡路"。
- 复制逻辑对"被复制对象"只依赖接口，不硬编码建筑类型。

### 2.6 边界与防坑
- 镜像落点约束见 §2.4b（在图外/被占 → 不生成）。
- **镜像可被另一面镜子再次复制（已确认 · 改动）**：镜像本身实现 `ICopyable`，可作为复制源被第二面镜子镜像，规则与复制建筑完全一致。**初版先做，出现性能问题（链式膨胀）再加上限**。
  - 防爆保护：预留 `copy_chain_max`（默认关闭/极大值），一旦性能吃紧改为小值截断链长。
  - 二级镜像仍遵守 §2.4b 占格与生成约束——落点被占则该级不生成，天然抑制无限膨胀。

## 三、反射镜 · ReflectMirror

### 3.1 行为定义
- 激光（由激光塔发射，固定方向的射线）打到反射镜的**生效面**时，按镜面反射定律改向。
- **单面**：只有生效面反射。
- 激光可被**多次反射**（不同入射角 × 不同镜朝向 = 不同光路）。

### 3.1b 背面光线处理（已确认 · 两种模式可选，编辑器切换）
反射镜背面被激光命中时的行为做成**可切换开关** `back_face_mode`，让设计者在编辑器里实测手感：
- `PASS`（穿过）：激光无视背面，继续直线传播。
- `BLOCK`（阻挡）：激光被背面挡停，**表现与"被地形阻挡"完全一致**（同一套截断/命中特效）。

**建筑不阻挡激光**（已确认）：激光穿过建筑格，只有障碍物 / 隆起地形 / 据点 / 边界 /（BLOCK 模式下的）镜子背面才会挡停。

### 3.2 激光传播规则（已确认）
- 激光沿固定方向直线传播，**无限传播**，直到遇到：
  - **反射镜生效面** → 反射改向（反射次数上限 `reflect_max`，默认 8，可配）。
  - **反射镜背面**（仅 `BLOCK` 模式）→ 挡停（表现同地形阻挡）。
  - **障碍物 / 隆起地形**（高于激光层的地形）→ 被遮挡，停止。
  - **我方据点 / 边界** → 停止。
  - **建筑**：不阻挡，激光穿过。
- 穿透语义：激光对路径上经过的**所有敌方单位**造成伤害（激光塔"无限穿透"指穿透单位，不穿透障碍/地形）。
- **伤害类型（已确认）**：激光对**触碰到的敌人施加持续伤害**（每帧/每 tick 按 `laser_dps` 结算，只要敌人还在光路上就持续掉血），而非一次性瞬伤。DPS 走标准伤害公式（固定值×等级因子×额外因子）。

### 3.3 反射几何
- 入射向量 `d`，镜面法线 `n`（生效侧）→ 反射向量 `r = d - 2(d·n)n`。
- 反射点 = 激光射线与镜面边（整条边即镜面）的交点。
- 每次反射后从交点沿 `r` 继续步进求交，直到命中终止条件或达 `reflect_max`。

### 3.3b 垂直入射的偏移处理（已确认）
- 当激光**垂直入射**反射镜（`d` 与法线共线）时，反射光会原路返回、与入射光完全重叠，视觉上"看不见反射"。
- 处理：反射光线起点沿镜面切向做一个**微小偏移** `perp_offset`（默认一个很小的定值），使**入射光与反射光平行但不重叠**（并排显示）。
- 仅垂直/近垂直入射时启用偏移；斜入射不需要（反射光天然分离）。

### 3.3c 端点命中判定（已确认）
- 镜子长度 = 边长（整条边就是镜面）。激光命中点落在这条边线段范围内即算命中，落在延长线上不算。
- 端点即线段两端：命中点若正好在端点上，按**"落点是否在镜面线段上"**的常规区间判定处理（`t ∈ [0,1]` 含端点即命中），无需特殊退化分支。

### 3.4 与激光塔的关系
- 激光塔：固定方向发射（塔朝向由 `R` 键调，见输入系统），激光=一条射线。
- 双重伤害：当反射光路与入射光路重叠某段时，该段敌人受两段激光叠加伤害（天然涌现，无需特判）。

### 3.5 边界与防坑
- 两面镜互相反射可能形成**闭环** → `reflect_max` 强制截断。
- 端点命中：见 §3.3c，按线段区间常规判定，无特殊分支。
- 激光每帧重算光路（塔/镜朝向可动），用射线步进 + 上限，性能可控。

## 四、参数表（全部 @export，编辑器运行时可调）

| 参数 | 归属 | 默认 | 说明 |
|---|---|---|---|
| mirror_cap | 资源系统 | 6 | 镜子总数上限 |
| mirror_cost | 资源系统 | 120 | 每面镜造价 |
| copy_recursion | CopyMirror | **true** | 镜像可否再被复制（已改为开启） |
| copy_chain_max | CopyMirror | 0(不限) | 复制链最大级数；出现性能问题时设为小值截断 |
| reflect_max | ReflectMirror | 8 | 单束激光最大反射次数 |
| back_face_mode | ReflectMirror | PASS | 背面命中模式：PASS 穿过 / BLOCK 阻挡(表现同地形) |
| laser_dps | ReflectMirror | 由塔种定 | 激光持续伤害每 tick 结算值(走标准伤害公式) |
| perp_offset | ReflectMirror | 0.05 | 垂直入射时反射光的微小平行偏移量 |
| laser_layer_height | ReflectMirror | 0 | 激光所在高度档(用于地形遮挡判定) |
| mirror_side_default | 通用 | + | 放置时默认生效侧 |

## 五、关键架构（占位，实现后补函数索引）
```
MirrorBase (Node)
 ├─ edge_ref: (tile, edge_index)
 ├─ active_side: enum(+,-)
 ├─ get_axis() -> Line          # 对称轴
 ├─ get_normal() -> Vector      # 生效侧法线
 ├─ CopyMirror
 │   └─ collect_copyables() -> [ICopyable]   # 依赖接口
 │       apply_mirror(transform)             # 点/向量关于轴镜像
 └─ ReflectMirror
     └─ reflect(ray) -> Ray
MirrorManager: 统一管理放置/查询/激光光路求解
```

## 已知限制
- 初版：只复制建筑（接口预留障碍物）；天空层不做。
- 镜像**可**再被复制（已开启），若链式膨胀致性能问题，用 `copy_chain_max` 截断。
- 美术：初版用**色块/点线面**替代（镜=线，塔/敌=几何体，激光=发光线段）。HexTileset 仅备用资产，未必采用。
