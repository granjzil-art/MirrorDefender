# 建筑系统 · Building

## 职责
定义可放置的防御建筑，初版为箭塔与激光塔，用组件/策略方式拼装以便扩展。

## 分类 / 做法
- **初版两种塔**：
  - **箭塔**：单体索敌攻击（索敌策略 + 单体攻击策略）。
  - **激光塔**：发射固定方向激光，无限穿透单位，可被反射镜反射（发射方向由塔朝向决定）。**对光路上触碰到的敌人施加持续伤害**（每 tick 按 `laser_dps` 结算，走标准伤害公式），非一次性瞬伤。激光**穿过建筑**、不被建筑阻挡。
- **组件/策略拼装**：建筑 = **索敌策略** + **攻击策略** + 数值配置。新增塔只需组合/新增策略，不改建筑基类。
- **镜像**：建筑可被复制镜镜像为**投影**（无独立 AI，行为完全复制原件，详见 Mirror）。
- **朝向（已确认 · 离散档位）**：`R` 键旋转建筑朝向；朝向档位**固定于世界坐标，不随镜头旋转变化**。
  - **六边形关卡：6 向**（始终垂直于六条边）。
  - **四边形关卡：8 向**（垂直于四条边 + 4 条对角线）。
  - 对激光塔，朝向决定激光发射方向。

## 关键参数
> 全部为 Godot `@export`，编辑器运行时可调。

| 参数名 | 默认值 | 说明 |
|---|---|---|
| damage | 10 | 基础伤害（战斗系统用作固定值项） |
| range | 3 | 攻击/索敌范围（格数）；激光塔为射线长度上限 |
| attack_speed | 1.0 | 每秒攻击次数 |
| cost | 100 | 建造消耗主资源 |
| level | 1 | 当前等级 |
| upgrade | [] | 升级路径配置（等级增伤因子等） |
| laser_dps | 激光塔专用 | 激光每 tick 持续伤害（走标准公式） |
| facing_slots | 6/8 | 朝向档位数：六边形 6、四边形 8（垂直边+对角线） |

## 关键架构
```
Building (Node)
 ├─ targeting_strategy: ITargeting   # 索敌策略（见 Combat）
 ├─ attack_strategy: IAttack         # 攻击策略（单体/激光）
 ├─ facing: 朝向（R 键旋转）
 └─ on_tick() → target = targeting.select(); attack.execute(target/dir)
ArrowTower  = Targeting(single) + Attack(projectile)
LaserTower  = Attack(laser_ray, facing)   # 交由 Mirror 求解光路
BuildingManager: 放置校验(依赖 Tile.can_place)、升级、复制镜投影生成
```

## 已知限制 / 初版不做的部分
- 初版仅箭塔、激光塔两种；不做范围/减速/增益类塔（extra_factor 预留）。
- 投影镜像无独立 AI，不可单独升级/点选。
- 不做建筑售卖返还的复杂比例，仅预留接口。
