## 1. 方案评估

### 1.1 亮点是否凸显？—— 凸显，但需要收敛

**镜子作为"动词"而不是"皮肤"，是这个方案最大的亮点。** 市面上叫 Mirror 的塔防大多只是把镜子当美术贴图，你这套把镜子拆成"边"和"面"两种几何角色，直接改变**空间关系**（复制、反射、跨层映射），这是真正的机制级创新，主题结合度很高。

风险在于**机制过载**：一次抛出三种差异很大的镜子玩法，玩家（和你自己的实现）都可能抓不住主线。建议确立一个**第一机制**作为"这游戏是什么"的记忆点，其余作为进阶层。我的推荐主线是 **边缘类·复制塔**——它最直观、最上手、最能体现"镜子 = 免费复制一份战力"的爽感。

### 1.2 与主题的结合程度

| 机制 | 主题贴合度 | 说明 |
|---|---|---|
| 复制塔 | ★★★★★ | 镜子最本能的联想就是"复制一个我"，几何上也自洽 |
| 折射双防 | ★★★★☆ | 光学反射是镜子的物理本质，非常正统，但需要"激光"这一额外系统支撑 |
| 基底映射 | ★★★☆☆ | 概念酷，但"倒影里的兵能打真实的敌人"需要一套自洽的世界观解释，否则玩家会困惑 |

### 1.3 不清晰 / 需要你拍板的点（这些我不擅自定）

1. **镜像单位是"实体"还是"投影"？** 决定了它能不能被攻击、会不会分摊仇恨、死亡是否联动。这是整套机制的地基，务必先定。
2. **复制是持续镜像还是一次性生成？** 持续镜像（本体动/死，镜像跟着动/死）与一次性复制（复制出独立个体）是两种完全不同的数值与代码模型。
3. **镜子占不占建造位？** 镜子架在"边"上，本身是否消耗资源/是否可被摧毁/敌人能否破坏镜子。
4. **双据点对攻的节奏**：是同时攻防的即时对抗（RTS 味），还是回合/波次分段（塔防味）？决定 3C 和 AI 复杂度。
5. **资源来源**：击杀掉落？占领地块产出？时间流逝自然增长？（影响扩张成就感的设计）

### 1.4 逻辑问题 / 一致性风险

- **复制的递归爆炸**：镜像出的单位如果又能被另一面镜子复制，会指数级增长。必须有规则封顶（如"镜像体不可再被镜像"，或复制有代数上限）。
- **折射光线的路径确定性**：激光遇到镜子反射，如果场上有多面可旋转镜子，光路会变成一个需要实时求解的几何问题，性能和可预测性都要评估。建议限制反射次数（如 ≤2 次）。
- **基底映射的"绕过"边界**：如果倒影单位能无视一切障碍，会破坏塔防的"路径规划"核心乐趣。需要定义清楚哪些障碍能绕、哪些不能。
- **镜像与索敌的仇恨归属**：镜像体吸引到的火力，本体是否受影响；镜像被摧毁本体是否掉血。

### 1.5 实现难点 & 容易踩坑的地方

| 难点 | 说明 | 建议 |
|---|---|---|
| 六边形坐标系 | 建造、索敌、寻路都要在六边形网格上做，不能用方格思维 | 采用**立方体坐标（cube coordinate）**，寻路/邻接/距离都有成熟公式，一劳永逸 |
| 镜像的"真实感" | 玩家期望镜像和本体动作严格同步，跨帧不同步会很出戏 | 镜像走"数据驱动 + 每帧同步 transform"，不要各自跑一套逻辑 |
| 折射的实时几何 | 可旋转镜子 + 光线反射是性能与确定性双坑 | 用射线步进 + 反射次数上限，参数全部可编辑器调 |
| 基底映射的相机与渲染 | "天上倒影层"需要第二套渲染或 Viewport，成本高 | 见 1.6，建议先做地面倒影，天空层作为可选特效 |
| 高低差与镜面反射一致性 | 见 2. 专题 | —— |

### 1.6 基底映射：地块镜子 vs 天上镜子（你问的取舍）

| 维度 | 地块镜子（地面水坑倒影） | 天上镜子（倒悬镜像层） |
|---|---|---|
| 实现成本 | 低：一个 Planar Reflection 平面 + 半透材质即可 | 高：需要独立的"上层"场景 / 第二 Viewport / 重力反转表现 |
| 主题表现 | ★★★★ 像雨后水坑，直觉、自然 | ★★★★★ 视觉震撼，"重力反转"记忆点强 |
| 玩法承载 | 弱：倒影挤在同一平面，单位容易重叠、难点选、难辨认 | 强：上下两层空间分离，倒影单位有独立落点，可做"上层通道" |
| 操作友好度 | 差：你自己也提到"实体镜像会挤在一起" | 好：两层错开，点选清晰 |
| 相机压力 | 小 | 大：要同时看清上下两层，缩放/旋转需要特殊处理 |

**我的结论与推荐：**
- **玩法层用"天上镜子"**——上下两层空间分离，才真正能承载"绕过河流/障碍"的策略深度，且点选不打架。
- **表现层同时保留"地面倒影"作为纯视觉特效**（不参与玩法），强化镜面主题氛围。
- 即：**逻辑上是天空层，视觉上地面也有一层不可交互的倒影**。两者不冲突，成本可控（地面倒影用便宜的 Planar Reflection）。

> 折中过渡方案：若首版想省成本，可先做"地下层"（把倒影层放到地平面**下方**，相机俯角天然能看到），比"天上层"少一套重力反转表现，且不遮挡主战场。

## 2. 专题：应该设计高低差吗？会不会冲淡镜面主题？

**结论：应该做，但要"服务于镜面"，而不是独立的地形系统。**

### 2.1 高低差的正价值
- 塔防经典价值：高台塔射程/视野加成，低地单位受压制，天然的地形博弈。
- 与你的资产吻合：KayKit 包里就有 `detail_hill` / `mountain`，本身支持高度表现。

### 2.2 会不会影响镜面主题？—— 处理得当反而**放大**主题
镜子的物理本质是"关于一个平面对称"。高低差恰好能让镜面对称"可见"：

- **水平镜面 + 高低差 = 高度翻转**：高台上的塔，其地面倒影（或天空镜像）会"往下/往上"翻折，高差越大翻折越明显，镜面感更强。
- **反例警告**：如果高低差是"随机地形起伏"，它和镜面没关系，就会显得杂乱、冲淡主题。所以规则是——**只在与镜面对称轴相关时引入高度**。

### 2.3 与镜子结合的高低差玩法（回答你"有结合可能吗"）
1. **高度镜像**：镜面不只镜像 XZ 平面位置，还镜像高度。地面高台的塔，天空镜像变成"从上往下打"的吊射塔，射程覆盖区不同。
2. **落差折射**：折射激光遇到高低差台阶时改变仰角，配合镜子做"立体光路"。
3. **镜像即桥**：低地被河流隔断时，天空镜像层是平的，镜像单位可从上层"跨过"河流，落到对岸——高低差制造障碍，镜子提供解法，形成完整的"问题—解法"闭环。

### 2.4 成本控制建议
高低差按**离散档位**做（如 0/1/2 三级），不做连续高度。离散档位对六边形网格、寻路、镜像计算都友好得多，也更符合塔防的"格子感"。
