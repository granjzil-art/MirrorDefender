# Mirror 塔防 · 策划设计案（系统框架）

> 目标：理清全部系统模块，覆盖传统塔防应有的一切系统（有哪些系统、有哪些参数、有哪些分类做法），**不深入实现细节**。
> 每个模块给出：职责 / 分类 / 关键参数 / 与镜子机制的关系。
> 版本：v0.1 · 2026-07-15 · `【待拍板】` 表示需你确认的默认值。

---

## 0. 游戏概览

| 项 | 内容 |
|---|---|
| 类型 | 3D 六边形网格塔防 + 双据点对攻 |
| 视角 | 斜俯视 3D，可 WASD 平移、QE 旋转、XC 缩放 |
| 胜负 | 守住己方据点（HP>0）+ 攻破敌方据点 HP=0 → 胜 |
| 核心动词 | 镜子：复制 / 反射 / 映射 |
| 技术栈 | Godot 4.x + GDScript，KayKit Medieval Hex 资产 |

系统总览（模块清单）：
`地块系统` · `坐标与网格` · `建筑系统` · `单位系统` · `资源系统` · `波次/战役系统` · `索敌与战斗` · `寻路系统` · `镜子机制系统` · `相机系统` · `操作/输入系统` · `UI/HUD 系统` · `AI 系统` · `存档/关卡系统` · `音效/表现系统`

## 1. 坐标与网格系统（Grid）

- **职责**：定义六边形网格的坐标、邻接、距离、方向。
- **分类做法（选型）**：
  - 立方体坐标 cube(x,y,z, x+y+z=0) — **推荐**，邻接/距离/寻路公式成熟
  - 轴向坐标 axial(q,r) — 存储省，可与 cube 互转
  - 朝向：pointy-top（尖顶）/ flat-top（平顶）— `【待拍板】`建议 flat-top（与你参考图一致）
- **关键参数**：`hex_size`(边长)、`orientation`、`grid_radius`(地图半径)、`tile_height_step`(高度档步长)。
- **可拓展点**：地块形状抽象为接口，未来可换三角形/正方形密铺（你的边界要求）。

## 2. 地块系统（Tile）

- **职责**：地图基本单元，承载地形、建筑、单位、镜子边、层归属。
- **分类**：
  - 按地形：平地 / 森林 / 山地 / 河流(不可通行) / 桥 / 据点地块 / 镜面地块
  - 按归属：己方 / 敌方 / 中立(可占领)
  - 按高度：0 / 1 / 2 三档（离散）`【待拍板】`
  - 按层：地面层 / 镜像层（天空或地下）
- **关键参数**：`terrain_type`、`owner`、`height_level`、`walkable`、`buildable`、`mirror_edges[6]`(六条边是否有镜子)、`capture_progress`。
- **与镜子关系**：地块的 6 条边可各自挂"边镜"；整块可设为"面镜"承载映射。

## 3. 建筑系统（Building）

- **职责**：可放置的防御/生产/功能性建筑。
- **分类（塔防标准分类）**：
  | 大类 | 举例 | 说明 |
  |---|---|---|
  | 攻击塔 | 箭塔/炮塔/激光发射器 | 单体/范围/穿透/减速等 |
  | 镜子建筑 | 复制镜/折射镜/单向镜 | 承载核心机制 |
  | 生产建筑 | 农场/伐木场/矿场 | 产资源（KayKit 有对应模型） |
  | 功能建筑 | 兵营(训练军队)/城墙/大门 | 阻挡/出兵 |
  | 据点 | 城堡 castle | 胜负目标，有 HP |
- **攻击塔关键参数**：`damage`、`range`、`attack_speed`、`projectile_speed`、`splash_radius`、`target_priority`、`damage_type`、`cost`、`upgrade_levels[]`、`sell_return`。
- **升级维度**：等级(数值成长) / 分支(如箭塔→穿透箭/多重箭)。
- **可拓展点**：塔行为用"组件/策略"拼装（索敌策略 + 攻击策略 + 效果策略），新塔=换组件。

## 4. 单位系统（Unit）

- **职责**：可移动的作战实体（含玩家训练的军队、敌方进攻单位、镜像体）。
- **分类**：
  - 阵营：己方 / 敌方 / 中立
  - 兵种：近战 / 远程 / 飞行(无视地面路径) / 重甲(高血慢速) / 突击(高速低血) / 特殊(碎镜者)
  - 实体性：**真实单位 / 镜像单位（投影 or 实体，`【待拍板】`）**
- **关键参数**：`max_hp`、`move_speed`、`armor`、`damage`、`attack_range`、`attack_speed`、`damage_type`、`faction`、`is_mirror`、`flying`、`layer`。
- **与镜子关系**：镜像体的 HP/伤害相对本体的系数(`mirror_stat_ratio`)、是否可再被镜像(`mirror_recursion_cap`)。

## 5. 资源系统（Resource）

- **职责**：建造/训练/升级的经济基础，驱动"扩张成就感"。
- **分类做法**：
  - 主资源（通用货币，如金币）— 建造升级
  - 次资源（木/矿，KayKit 生产链）— `【待拍板】`是否引入多资源
  - 人口/军队上限
- **产出来源**：击杀掉落 / 占领地块被动产出 / 生产建筑 / 时间自然增长（推荐"占领产出"以强化扩张感）。
- **关键参数**：`gold`、`wood`、`stone`、`population/pop_cap`、`income_per_sec`、各建造/升级`cost`。

## 6. 波次 / 战役系统（Wave / Campaign）

- **职责**：组织敌方进攻节奏与关卡目标。
- **分类**：
  - 波次制：一波波刷怪，波间有准备期（经典塔防）
  - 对攻制：双方持续出兵互攻据点（你的核心）
  - 战役：多关卡串联，带剧情/解锁
- **关键参数**：`wave_list`(每波构成/数量/间隔)、`spawn_points`、`prep_time`、`difficulty_curve`、`victory_condition`、`defeat_condition`。

## 7. 索敌与战斗系统（Targeting & Combat）

- **职责**：塔/单位如何选择目标、如何结算伤害。
- **索敌策略分类（塔防标准）**：最近 / 最远 / 最高血 / 最低血 / 最快 / 优先飞行 / 首个进入(First) / 锁定不换。
- **伤害类型与克制**：物理/穿透/魔法/灼烧 × 无甲/轻甲/重甲/护盾（矩阵克制表）。
- **关键参数**：`target_priority`、`can_target_air`、`damage_type`、`armor_type`、`crit`、`dot`(持续伤害)、`slow/stun`(控制)。
- **与镜子关系**：镜像体是否进入索敌池、镜像诱饵如何操纵仇恨。

## 8. 寻路系统（Pathfinding）

- **职责**：单位在六边形网格上的移动路径。
- **分类做法**：A*（六边形邻接）/ 流场 FlowField（大量单位时更优）`【待拍板】`。
- **关键参数**：`path_grid`、`obstacle_mask`、`layer`(分层寻路，地面/镜像层各一张图)、`repath_interval`。
- **与镜子关系**：镜像/传送门改变连通性，寻路要支持跨层。

## 9. 镜子机制系统（Mirror）★核心

- **职责**：统一管理三类镜子行为，作为独立可插拔模块。
- **子模块**：
  - 边镜·复制：`copy_mode`(持续/一次性)、`recursion_cap`、`stat_ratio`
  - 边镜·折射：`reflect_count_max`、`beam_damage`、`beam_width`、镜子`angle`(可运行时调)
  - 面镜·映射：`mirror_layer`(天空/地下)、`mapped_bypass_mask`(能绕过哪些障碍)
- **全参数化**：所有系数暴露到编辑器（你的硬性要求）。
- **可拓展点**：新镜子类型 = 实现同一 `IMirrorEffect` 接口。

## 10. 相机系统（Camera）

- **职责**：呈现战局与扩张感。
- **参数**：`move_speed`(WASD)、`rotate_speed`(QE)、`zoom_min/max`(XC)、`pitch_angle`(俯角)、`edge_pan`、`focus_smooth`。
- **与镜子关系**：双层战场时的分层可见性/半透过渡。

## 11. 操作 / 输入系统（Input）

- **职责**：3C 中的 Control。映射你指定的操作方案。
- **映射表**：WASD 平移 · QE 旋转 · XC 缩放 · 鼠标左键选择/放置/确认 · 鼠标右键取消/信息 · 悬停高亮。
- **参数**：全部键位做成可配置 `InputMap`（可拓展/可改键）。

## 12. UI / HUD 系统

- **职责**：信息呈现与操作入口。
- **模块**：顶部资源栏 / 底部建造栏(卡片) / 选中单位信息面板 / 升级-出售面板 / 波次进度条 / 双据点血量条 / 小地图(含双层切换) / 镜子放置预览。
- **参数**：布局锚点、缩放适配、可读性配色（据点血量、敌我色）。

## 13. AI 系统

- **职责**：敌方据点的进攻/防守决策。
- **分类**：脚本波次 AI（简单）/ 行为树（中）/ 效用 AI（自适应难度）`【待拍板】`。
- **参数**：`aggression`、`economy_focus`、`target_selection`、`difficulty`。

## 14. 存档 / 关卡系统

- **职责**：关卡数据、进度、配置的加载与保存。
- **做法**：关卡用数据资源(.tres/JSON)描述地图与波次，与逻辑解耦（可拓展新关卡不改代码）。

## 15. 表现 / 音效系统（FX/Audio）

- **职责**：镜面反射特效、激光、命中、UI 音效、BGM。
- **与镜子关系**：地面倒影(Planar Reflection)、镜像同步动画、折射光束 VFX。
