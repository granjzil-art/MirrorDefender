# 技术与玩法决策记录

## 2026-07-23 · 六机位使用可空关卡子资源与独立真实时间控制器

**决策**：每个 `LevelResource` 以最多 6 个可空 `CameraPresetDefinition` 子资源保存焦点世界坐标、yaw、pitch 和缩放距离；空数组是旧关卡兼容表示，不在加载或打开编辑器时补齐。`CameraPresetController` 只读取当前关卡，通过 InputMap 的数字键动作驱动 `CameraController`，用默认 0.35 秒真实时间和最短 yaw 角插值；过渡期间只抑制手动镜头输入，暂停/控制台模态沿用相机输入总锁并冻结过渡。

**编辑器决策**：镜头编辑使用独立“镜头”页、`CameraPresetEditor` 侧栏和专用 `TileEditorCanvas`，不复用地块页选择状态。编辑器正交像素比例按运行时默认 75° 垂直 FOV 转换为缩放距离；保存的事实始终是运行时距离而不是编辑器像素值。

**理由**：可空子资源让每个槽独立配置和验证，并保持旧关卡零迁移；独立控制器避免把数字键、插值和关卡生命周期继续堆进 gimbal 相机或 `Main.gd`。专用编辑画布隔离地块刷、路径记录与镜头预览状态。

**约束**：槽位固定为 1～6，未配置或无效槽位无动作；切关取消未完成过渡；手动 WASD/QE/XC/滚轮在过渡完成后继续生效。若未来运行时 Camera3D 改变默认 FOV，必须同步编辑器换算参数或将 FOV 纳入统一相机配置。

## 2026-07-22 · 独立关卡端点、多据点共享生命与目标锁定换路

**决策**：出生点从“每路径 1:1 派生”改为独立关卡对象，可被多条路径共用。据点也改为独立位置数组，每条路径显式选择一个出生点和一个目标据点；多个据点位置共用一份 BaseCore 生命。本决策明确替代 2026-07-19 的“路径/出生点 1:1”新数据约束，但保留其旧关卡只读兼容策略。

**寻路决策**：敌人生成时锁定初始路径的目标据点。大石头阻挡时，先使用同目标据点的最短可用手工后缀；无完整后缀时，在“同目标手工路径格并集”上执行可替换策略的确定性 A*。不允许转去其他据点，不允许在手工路网外自由行走；仍无路时攻击当前大石头。普通屏障/边障继续直接攻击。

**兼容与编辑边界**：旧 `base_cell` 在 `base_points` 为空时映射为不写回资源的“据点 1”；旧路径通过显式引用、波次引用或首/末格唯一匹配解析端点。编辑器加载不做迁移写入；只在用户主动编辑据点时物化 `base_points`。出生点与据点的数字同时显示在编辑器和运行时世界。

## 2026-07-22 · M6 采用模块化正式 HUD 与单次放置交互

**决策**：当前 M6 调整为“操作与 UI 大版本”，旧版计划中的反射镜里程碑顺延待排期。正式运行时采用模块化 HUD、`RuntimeInteractionController` 和 `GameTimeController`，不继续让 `Main.gd` 或开发期 `M3DebugPanel` 承担具体 UI 状态。整体固定为左侧纵向波次时间轴、底部独立镜子槽加单行建筑卡槽、右上全局信息、右侧地块详情、右下经济/时间/暂停，模态层承载暂停菜单和 F1 调试控制台。

**交互契约**：左键执行全部肯定操作；右键只取消。每次选卡只允许一次放置尝试，成功、资源不足、达到上限、位置非法或未命中都会取消卡片并回选择模式，新建筑不自动被选中。默认携带 `箭塔 / 激光塔 / 屏障`，建筑槽默认 6 个；复制镜使用额外的独立槽。选卡或选中实体建筑/边建筑/镜子触发默认 0.1 倍战术慢放，优先级为 `暂停 0x > 慢放 0.1x > 快速 2x > 正常 1x`。

**理由**：正式 UI 同时涉及交互、时间、经济、波次、关卡编辑器与调试入口。如果继续扩张开发期面板，会让输入消费、切关生命周期和玩法状态形成循环依赖。单次尝试契约让成功与失败路径一致，也避免玩家因一次失败意外连续建造。

**长期事实源**：`Docs/07_M6_操作与UI大版本_需求与开发计划.md` 固定全部布局、默认值、模块边界、六个开发批次和总体验收标准；本决策不改变既有玩法与数值，边障保留但不进入默认卡组。

## 2026-07-21 · 黑洞以真实源格共享周期与容量状态

**决策**：黑洞不再于敌人进格时立即触发，而是每个真实源格拥有独立的装填量、逐点恢复时钟和周期吞噬时钟。检查时有容量则吞噬格上当前生命最高的一个适用敌人。直接/递归投影不创建独立容量，而是通过根源格状态键共享原黑洞的全部运行时状态与坑洞深度。

**理由**：全局周期检查才能让快速经过的敌人有可能逃脱，而格上集中选择能保证多敌人同时经过时严格按当前生命决胜。复制镜的“纯复制虚像”语义要求投影反映源状态，否则复制一个黑洞会凭空创造额外容量。

**运行时边界**：状态仅存在当前关卡运行时，切换/重载关卡清零；不写回 LevelResource 或共享 TileEffect 资源。`affects_airborne` 仍在候选收集前过滤飞行单位。

## 2026-07-21 · 普通屏障直接攻击，大石头先换路后共享源承伤

**决策**：路径阻挡只分两种受阻响应。屏障、边障采用 `DIRECT_ATTACK`，敌人直接停步攻击；大石头采用 `REROUTE_THEN_ATTACK`，仅在前一格尝试其它手工路径，找到最短可用后缀则换路，没有候选才攻击当前石头。真实石头按格拥有运行时耐久；石头直接/递归投影与真实源共享该耐久，投影承伤回传根源。源石头摧毁后清除内容和导航阻挡、开放块建筑与边建筑，投影消失但镜子保留。

**理由**：两类障碍的差异只应存在于受阻决策，攻击射程、攻速、近战/投射物和结构伤害可复用同一套流程。把当前耐久保存在共享 `RockTileEffect` 会使所有引用同一资源的石头串血；逐格运行时节点既保留关卡元素的编辑器涂刷语义，也能为镜像提供唯一真实伤害源。

**否决的备选**：

- 大石头永久不可攻击、无路原地等待：会造成永久卡死，不符合最新玩法。
- 无路等待固定时间后穿越：不再符合“大石头无路时等同普通可破坏障碍”的规则。
- 每个石头虚像拥有独立耐久并在归零时击碎镜子：破坏共享源复制语义，也让递归镜链产生额外生命归属。
- 对普通屏障也进行换路：削弱屏障直接阻挡价值，并改变现有玩法。

**运行时边界**：当前耐久、清障状态和建筑占位不写回 LevelResource；重新加载关卡恢复石头。大石头不进入建筑上限、升级、退款和资源掉落流程。`affects_airborne` 继续决定石头是否阻挡并可被对应敌人攻击。

## 2026-07-21 · 逻辑朝向、视觉追踪和投影姿态分层

**决策**：建筑的 `facing_index` 始终是放置与固定方向攻击的逻辑事实源。`BuildingDefinition.AimMode` 声明转向能力：`FIXED_FACING` 的激光类建筑只跟随手动逻辑朝向；`TRACK_TARGET` 只转动 `_visual_root` 跟随已锁定目标，不改写 `facing_index` 或攻击判定。镜像投影保留一个无行为快照，每帧同步源子节点与骨骼姿态，并使用 `payload.transform_transform(source_visual_transform)` 完成全模型的组合镜像。

**理由**：按建筑种类在运行时写死转向会阻碍后续追踪塔变种；直接旋转 Building 根节点又会破坏激光线段和手动放置朝向。只在源姿态改变时重建投影会重现透明几何闪烁，而只复制 yaw 无法镜像非对称模型或子节点动画。

**约束**：投影仍无独立索敌、冷却和动画时钟；投影材质保留虚像辨识特征，几何变换与姿态则严格源自实体后再作用镜面反射。

## 2026-07-21 · 实时镜面补偿反射相机的横向手性

**决策**：保留镜像主相机与屏幕对齐反射纹理。虚拟相机使用右手基底重建反射后的 forward/up，其屏幕 X 与主相机手性相反；Shader 因此使用 `vec2(1.0 - SCREEN_UV.x, SCREEN_UV.y)` 读取对应反射像素。验收语义是镜前右侧地块被选中时，镜中仍是右侧地块被选中。

**理由**：不补偿时会出现用户观察到的右→左横向交换。因为反射目标与主视口使用相同投影和宽高比，手性差恰好是全视口屏幕 X 取反；围绕单面镜子的局部屏幕中心翻转反而会在镜子偏离画面中心时读取错误区域。此补偿不改复制镜的玩法生效侧或目标格计算。

## 2026-07-21 · 健壮性测试与正式调参资源解耦

**决策**：机制测试使用 `tests/fixtures` 的显式内存定义；策划可编辑的正式 `.tres` 只验证类型、可加载性和 `validate_configuration()`，不再断言具体平衡值。统一测试入口除进程退出码外还扫描 Godot 的脚本错误、引擎错误和泄漏警告。共享 ConfigurationValidator 只返回错误数组，不修复或钳制资源。

**理由**：镜子高度、双面显示和 M4 波次延迟属于合法调参，旧测试把这些资源快照当成代码契约，导致功能正常时基线仍为红色；Godot 测试脚本也可能在记录 SCRIPT ERROR 后继续执行 `quit(0)`。稳定夹具隔离机制契约，正式资源冒烟则保留资产损坏防线。

## 2026-07-21 · 关卡装配失败回滚并按真实地块顶面拾取

**决策**：LevelLoader 在预检后保存旧 Grid 配置；若 TileManager 仍拒绝安装，则恢复 Grid，保留旧 Tile 字典与 current level。运行时拾取不再把地图简化为 `Y=0` 平面，而是通过 Main 注入的只读高度查询，与全部地块水平顶面求交并选择射线最近结果；GridManager 不持有 TileManager。

**理由**：预检降低失败概率但不能代替事务边界，未来 TileManager 增加装配条件时可能在 Grid 已变化后拒绝。固定地面射线在斜视角和高地上会穿过可见顶面、选中后方低地。回滚保持关卡原子性，Callable 高度端口同时保留 Grid/Tile 模块方向。

## 2026-07-21 · 复制镜玩法生效侧与视觉观察侧解耦

**决策**：复制镜继续用 `active_from_side` 作为最近源格、目标格、投影方向和顶部标识的唯一玩法事实源；实时镜面默认允许从两侧观察，但始终只维护一个反射 Quad、一个 SubViewport 和一个反射相机。刷新时反射 Quad 贴到主相机所在的实体表面；虚拟相机严格镜像主相机姿态与投影，渲染目标匹配主视口宽高比，镜面通过 `SCREEN_UV` 采样。镜面与实体背板使用反射相机排除层，避免递归和自身背板遮满纹理。`reflection_two_sided_visual` 可恢复旧的单生效侧表现。

**理由**：旧的法线对齐离轴相机在拉远后会产生远大于镜面尺寸的 frustum offset，Godot 最终只输出清屏色；相机跨过某面镜子的无限平面时，还会触发正面候选拒绝和背面剔除。提高 Camera `Far` 无法改变这两类问题。屏幕对齐投影消除距离相关的极端 frustum，视觉侧解耦则保持跨侧连续，两者都不污染复制玩法。

**性能边界**：不为背面创建第二套渲染目标；MirrorManager 原有刷新间隔与每帧上限继续生效。可见性使用镜面中心加四角采样，仅视锥外镜子停止刷新。反射分辨率仍表示水平像素数，垂直像素数按主视口宽高比自动计算。

## 2026-07-20 · 关卡下方倒影采用单共享世界平面反射

**决策**：关卡下方只创建一个水平 `PlaneMesh + SubViewport + Camera3D`。虚拟相机视点为主相机关于反射平面的实时对称点，离轴视锥严格匹配关卡包围矩形；水面材质只对实时纹理做柔化、Fresnel 和程序化雨滴扰动。该节点属于 FX 表现层，不创建碰撞、地块占位或任何玩法注册。

**理由**：固定投影、截图贴片和预烘焙环境图无法随主相机、单位和关卡状态变化；复制整套关卡节点又会复制脚本状态并污染玩法。共享 `World3D` 的单平面反射能保持动态画面，同时把性能控制收敛为一个分辨率和刷新间隔。

**渲染边界**：复制镜面与关卡水面共用隔离可见层 20，所有反射相机都剔除该层，因此不会发生镜中水面、水面中镜面或多视口递归。连续不透明地形会按真实遮挡隐藏正下方倒影，不用抬高水面覆盖地块来制造假反射。

**否决的备选**：

- 固定 RenderTexture / 屏幕截图：成本低，但旋转视角和动态单位会立即穿帮。
- 复制并上下翻转全部关卡节点：易于看到倒影，但会重复脚本、动画、粒子和玩法生命周期。
- 在每个地块顶面覆盖透明水膜：水坑更明显，但会改变全部地块颜色和深度关系，不符合“关卡下方”的范围。

## 2026-07-20 · 复制镜只复制地块内容，地形基底属于目标关卡

**决策**：“复制最近非空格内容”中的内容只包含格建筑、石头/尖刺/空洞等地块元素及既有投影；地表顶面、侧壁、高度色、路径色和路面色不进入投影快照。元素几何仍按镜轴做严格对称，不用替代几何或偏移。本决策取代“地块投影包含完整地表”的旧表现约束。

**理由**：地形基底是目标格的关卡信息，复制源基底会遮蔽目标格的路径/高度语义，并与已有地表生成共面深度冲突。只复制内容同时保留玩法效果、严格对称外形和目标地形可读性。

**渲染约束**：同格透明投影使用稳定 `render_priority` 且不写深度；重建先隐藏旧节点再延迟释放，防止同帧重叠闪烁。

## 2026-07-20 · 路径颜色属于地块基底，地块内容使用独立表现层

**决策**：全部手工敌人路径经过格的并集使用 `LevelResource.path_terrain_color`，默认 `#FFB93B`；该优先级高于路面、高度色和单格颜色覆盖。建筑 occupant 和 `SurfaceKind.ELEMENT` 的石头/尖刺/空洞不参与基底颜色解析；元素只用 `visual_color` 或 `visual_scene` 绘制上层内容。运行时、关卡编辑器与镜像地块快照共用该规则。

**理由**：路径是关卡导航语义，不应因为同格放置建筑、陷阱或障碍而失去可读性。把基底与内容几何分开，也让未来新元素只需实现自身视觉契约，不会意外改写关卡地形信息。

**否决的备选**：继续让地块元素通过 `terrain_color` 覆盖整个顶面，会隐藏路径与高度基底；仅依赖 PathManager 的悬浮调试线，则无法在关闭调试表现后继续表达路径地形。

## 2026-07-20 · 复制镜表现采用共享世界平面反射与真实源快照

> 其中“法线对齐离轴视锥”已被 2026-07-21 的屏幕对齐反射决策取代；共享 `World3D`、单视口预算和真实源快照约束继续有效。

**决策**：复制镜的生效面用共享主场景 `World3D` 的 SubViewport 显示完整关卡反射，虚拟相机位置关于镜轴反射，并以实体镜面矩形建立离轴视锥。虚像必须复制源 Building 或 TileRenderer 的实际视觉快照，再应用 payload 全部镜轴组成的严格仿射反射；地块投影包含正常地表、真实高度、侧壁、颜色及元素。重叠虚像不得位移、缩放、错层或用独立几何替代，只能用材质轮廓、不同强调色、同心标识和悬停信息辅助辨认。

**理由**：屏幕空间假镜面或只渲染单个建筑无法表达“镜中完整关卡世界”；按类型造灰盒会让尖刺/空洞/岩石和未来美术资产失真；位移拆分又会破坏“原实体关于镜轴对称”的核心视觉规则。共享世界离轴视锥与源渲染快照把玩法变换和表现变换统一到同一事实源。

**性能边界**：反射面使用独立可见层并从所有反射相机中排除，不支持无限镜中镜；MirrorManager 只轮询正面可见镜子，默认每 2 帧最多刷新 2 面，正式/预览分辨率分别为 256/128。虚像快照无脚本、音频和独立动画时钟，攻击表现继续由原件事件同步。

**否决的备选**：

- 屏幕截图或静态环境贴图：成本低，但镜面不随相机、敌人和关卡状态正确变化。
- 每种 `copy_kind` 独立制作简化几何：便于灰盒，却不能保证地块和正式资产与原件一致。
- 将重叠虚像扇形展开、缩小或垂直错层：易辨认，但破坏严格对称位置和尺寸。
- 递归渲染其它镜面：物理感更强，但多镜时渲染量指数增长并容易形成反馈纹理。

## 2026-07-20 · 复制镜采用最近整格、非占位投影覆盖层

**决策**：复制镜是共享物理边占用的边建筑。它从生效侧相邻格开始沿镜面法线向外扫描，只选择最近的一个含可复制内容的源格，并将该格全部格建筑、地块元素和既有投影镜像到另一侧等距格；镜子、其它边建筑、敌人和纯地形数据不复制。投影默认不占实体地块 occupant、允许同格叠加和落在不可建造地块，`projection_ignores_occupancy` 可切换严格占位；镜链默认最多 4 层且同一链禁止重复经过同一面镜子。

**理由**：最近整格规则能稳定表达参考图中的一镜一来源，同时保留“一格组合机关整体复制”的设计空间。独立投影覆盖层既不破坏关卡原始 TileCellData，也能让塔、屏障、尖刺、空洞和岩石分别接入战斗、阻挡、地块效果与导航查询。有限谱系使投影可再次复制而不会形成无限镜环。

**玩法约束**：塔投影无独立索敌、冷却或动画，只同步镜像原塔攻击并允许打空；屏障投影承伤转发原屏障；地块元素继承源资源和 `affects_airborne`。源格内容改变时动态重算。放置预览必须显示生效侧、源格、目标格、内容种类与投影虚影；无源或越界只警告，不禁止放置。

**否决的备选**：

- 只复制镜边相邻建筑：操作直观，但无法实现隔空复制和参考图中的最近非空玩法。
- 克隆真实 Building / 修改目标 TileCellData：复用逻辑较快，但会错误引入独立 AI、经济、占位和存档污染。
- 投影默认占格：冲突少，但无法投到石头、尖刺、空洞，也不能实现同格多镜叠加。
- 无限制递归：规则最自由，但镜环会造成不可控膨胀和帧率依赖。

## 2026-07-20 · 动态换路只识别导航阻碍

**决策**：候选手工路径在接入关系、边界与逐格相邻合法的前提下，唯一的地块过滤规则是“后缀不含导航阻碍”。空洞和尖刺均为可通行地块，可被动态换路选中，其进入/停留效果在敌人实际通过时结算。本决策取代 2026-07-19 决策中“后缀不含空洞”的部分。

**理由**：换路系统只负责导航可达性，不预判地块危害；这使空洞陷阱能对换路敌人生效，也避免再引入一套与导航阻断并行的“安全路径”参数。

## 2026-07-19 · 关卡元素用地块策略，岩石换路限于手工路径

**决策**：尖刺、空洞和大石头是 `TileDefinition + TileEffect` 配置，不是 Building。大石头只在敌人抵达前一格时触发一次换路选择：候选必须是其他手工路径，与当前格相交或相邻，后缀逐格可达；按接入边与后缀边数之和选最短，平分按关卡路径顺序。地块过滤规则已由 2026-07-20 决策更新。

**理由**：关卡元素需要在编辑器直接涂刷，不应引入建筑经济、等级和占位生命周期。地块策略将通行、进入/停留效果与表现数据分离，新变种可通过资源扩展。仅在手工路径中换路可保留关卡设计者对行军线的控制，并使 HEX/SQUARE 只依赖 GridManager 的相邻事实。

**否决的备选**：

- 把三种元素做成 Building：可复用节点表现，但会错误引入玩家建造、上限、升级和退款语义。
- 对所有可通行格运行 A*：能找到更多路，但会让敌人脱离手工路径，与“路径仍手工指定”的关卡设计目标冲突。
- 见到前方岩石就提前换路：观感更智能，但不符合“只有到达阻碍前一格才触发”的规则。
- 一次搜索串联多条候选路径：可达性更高，但候选图和平分语义过早复杂化；当前每次只转入一条路径后缀，之后遇到新障碍再次触发。

**兼容与约束**：旧 `TileCellData.tile_type` 在 `definition == null` 时保留原行为，加载时不回写旧 `.tres`。三种新元素默认禁止块建筑且允许边建筑，共享边需两侧都允许。建筑屏障继续可攻击，不属于导航阻断。

## 2026-07-19 · 路径是出生点与波次选择的事实源

**决策**：一条路径只对应一个出生点。新建数据以 `path_N -> spawn_path_N` 派生 ID，显示名为“路径名 + 出生点”；波次编辑只选路径，出生点随路径自动绑定。

**理由**：敌人的路径首格就是其出生格，允许两个独立下拉选项会制造无效组合；统一标签与稳定 ID 同时解决子资源被显示为关卡文件名的问题。

**否决的备选**：

- 保留路径/出生点两套独立编号与下拉选择：能表达出生点复用，但与当前 1:1 玩法约束冲突，且容易误配。
- 以数组下标隐式绑定：实现短，但删除、重排或旧关卡导入时不稳定。

**兼容策略**：不在加载时改写旧 `.tres`；综合新派生 ID、旧 SpawnGroup 的成对引用与起点格查找候选。候选唯一才绑定；候选歧义则提示用户清理，不猜测也不新建重复项。只有用户主动同步或编辑路径时才把唯一旧出生点转换为新命名。

## 2026-07-19 · 边屏障改为任意内部边放置并默认双向

**决策**：边屏障不再依赖关卡路径缓存；任意两个地图内有效地块之间的共享边均可放置，出生点/据点相邻边不例外。阻挡以方向无关的 `canonical_edge_id` 查询，`blocks_both_directions` 默认 `true`；关闭时才回退到放置视角记录的单向匹配。

**理由**：边障应是可先于路径规划存在的通用关卡构件，路径只在敌人实际穿越该边时触发它。默认双向更符合墙体直觉，也消除玩家必须从特定一侧精确选边的操作负担。

**否决的备选**：

- 保持只允许路径边：规则约束清晰，但无法预先布置，也限制关卡编辑自由度。
- 允许地图最外圈无邻格边：视觉上更自由，但没有第二个有效格、不会被任何合法路径穿越，且双向语义不成立。

**约束**：同一物理边仍只允许一个边建筑；敌人当前位于任一相邻格时拒绝建造；默认双向参数可在 Definition 的 Placement 分组修改。本决策取代上一条“反向默认不阻挡”和“出生点/据点相邻边不可放置”的约束。

## 2026-07-19 · 边屏障分离物理边占位与有向路径阻挡

**决策**：关卡几何标签由 `LevelResource.grid_shape` 派生，不保存重复字段。边建筑以方向无关的 `canonical_edge_id` 保证一条物理边最多一个占位，同时以方向相关的 `directed_edge_id(from_cell, to_cell)` 决定阻挡；边屏障只匹配放置时从鼠标所在格指向邻格的路径前进段，反向默认不阻挡。EnemyUnit 通过注入的二参数解析器逐段查询，不依赖 Building 类型。

**理由**：物理占位和玩法方向是两个不同事实；只用物理边键会错误阻挡反向路线，只用有向键则允许两个建筑重叠在同一条边。标签从几何派生可杜绝“标记为六边形但实际加载正方形”的非法组合。逐段注入查询也能让地块屏障与边屏障共享敌人攻击流程，而不引入动态寻路。

**否决的备选**：

- 以路径 ID 绑定边屏障：能隔离完全重合路径，但需要放置时额外选择路径，且同向共享道路会产生不直观的穿透。
- 同一物理边双向都阻挡：规则简单，但不符合参考图的方向性，也无法表达单向关卡机关。
- 新增独立可编辑“关卡类型”字段：Inspector 更显眼，但会与 `grid_shape` 形成两份可冲突事实源。

**约束**：共享同一有向格段的路径共同受阻；边建筑放置后不能自由旋转；出生点/据点相邻边不可放置；后续镜面等边建筑复用物理占位键，但自行定义方向效果。

## 2026-07-19 · 关卡资源只作配置快照，运行时安装必须先校验后提交

**决策**：LevelResource 及其 TileCellData 只作为可保存配置；LevelLoader 在任何 Grid/Tile 状态变化前执行只读 `validate_runtime()`，TileManager 仅在完整构造下一份字典后一次替换当前状态，并为每个序列化地块创建独立运行时副本。波次依赖或生成失败进入显式 `CONFIG_ERROR`，不得把失败生成视作已完成。

**理由**：Resource 会被 Godot 缓存，也可能被编辑器和多个运行实例同时引用。把 occupant、清障或局内高度直接写回 Resource 会造成跨关卡/跨实例污染；先改 Grid 再发现 Tile 无效会留下半装配状态；静默跳过失败敌人则可能耗尽计数并错误判胜。

**否决的备选**：

- 继续复用序列化 TileCellData 作为运行时对象：对象引用最少，但局内状态会污染缓存和其它实例。
- 加载时边校验边修改各管理器：实现较短，但任一中途失败都需要跨模块回滚。
- 出怪失败只打印错误并扣除剩余数量：运行不中断，但胜利条件失真，配置错误难以定位。

**约束**：预检必须只读；失败加载保留当前关卡；TileManager 直接加载时要求 Grid 已与 LevelResource 配置一致；运行时变化不写回 `.tres`；编辑器仍可经二次确认保存未完成关卡，但运行时不会安装它；本决策不改变任何玩法数值。

## 2026-07-19 · 屏障作为路径专用 Building，并以注入查询驱动敌人攻击

**决策**：屏障复用 BuildingDefinition、BuildingLevelStats、BuildingManager、Tile occupant、升级和经济事务；普通塔禁止占据路径格。EnemyUnit 不直接依赖 Building 类型，而是通过 Main 注入的 `Callable(BuildingManager.get_path_blocker)` 按 PathDefinition 顺序查询前方屏障，并依赖结构方法契约造成伤害。

**理由**：屏障需要复用现有建筑的放置、升级、删除、资源和选择 UI，同时又不能进入 CombatManager 的敌方 CombatTarget 候选。注入查询保留 Unit/Building 模块边界；路径仍是设计者静态事实源，无需为可摧毁屏障引入动态寻路。

**否决的备选**：

- 将屏障继承 CombatTarget：会让我方塔把我方屏障加入敌对索敌候选，阵营过滤尚不存在。
- 将屏障做成独立于 Building 的新系统：会重复经济、占位、升级、退款和 UI 流程。
- 屏障出现后动态重算路径：与“阻挡后攻击、摧毁后继续原路线”的需求不符，也会提前引入导航系统复杂度。

**约束**：出生点和据点不可放屏障；未清障格不可特殊占位；屏障战斗摧毁不退款，玩家主动删除按等级退款；所有敌人在攻击状态停止移动。

> 本条中的单参数 `get_path_blocker(cell)` 继续描述地块屏障；2026-07-19 的边屏障扩展把 EnemyUnit 注入入口升级为 `resolve_path_blocker(from_cell, to_cell)`，并在内部兼容查询终点地块屏障。
## 2026-07-20 · 效果拥有空中适用性，目标只拥有分类

**决策**：EnemyDefinition / CombatTarget 只定义 `is_airborne` / `airborne` 类别事实；TileEffect 和每级 BuildingLevelStats 各自通过 `affects_airborne` 决定效果是否对空中目标生效。EnemyUnit 把自身作为可选 target 传入地块导航、换路和建筑阻挡解析，攻击策略使用 Building 的统一过滤入口。

**理由**：“飞行”是目标属性，但“能否命中飞行”是效果属性。将免疫表写在 EnemyUnit 会让新地块/建筑变种持续修改单位模块；让策略各自判定则易产生箭塔、激光和屏障行为不一致。

**兼容策略**：新增 `affects_airborne` 默认 `true`，旧 `.tres` 缺少字段时保持原有效果。调用解析器时 target 为可选参数，旧直接查询仍按原行为返回。飞行单位仍使用波次手工路径，本决策不引入自由空域寻路。
